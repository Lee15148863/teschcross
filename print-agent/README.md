# Tech Cross Print Agent

Local print agent for the Tech Cross POS system. Runs on the till computer and receives receipt data from the browser-based POS page via HTTP, then prints to an 80mm thermal printer using ESC/POS commands.

## Prerequisites

- Node.js 18 or later
- An 80mm ESC/POS compatible thermal printer (e.g. MUNBYN ITPP047)
- Network or USB connection to the printer

### USB printing (optional)

If you plan to use USB printing, you also need:

- **Windows**: Zadig driver (WinUSB) for the printer
- **macOS / Linux**: libusb (`brew install libusb` on macOS, `sudo apt install libusb-1.0-0-dev` on Ubuntu)

## Setup

1. Install dependencies:

   ```bash
   cd print-agent
   npm install
   ```

   > **Note:** The `escpos-usb` package requires native USB libraries. If you only use network printing you can skip USB support by removing `escpos-usb` from `package.json` before installing.

2. Edit `config.json` to match your printer setup:

   ```json
   {
     "port": 9100,
     "printer": {
       "type": "network",
       "host": "192.168.1.100",
       "port": 9100
     },
     "usbFallback": {
       "vendorId": "0x0416",
       "productId": "0x5011"
     },
     "logoPath": "../teschcross/logo.png"
   }
   ```

   | Field | Description |
   |-------|-------------|
   | `port` | HTTP port the agent listens on (default 9100) |
   | `printer.type` | `network` (recommended) or `usb` |
   | `printer.host` | Printer IP address on the local network |
   | `printer.port` | Printer port (usually 9100) |
   | `usbFallback.vendorId` | USB Vendor ID (hex) — used when network is unavailable |
   | `usbFallback.productId` | USB Product ID (hex) |
   | `logoPath` | Relative path to the company logo PNG |

3. Start the agent:

   ```bash
   npm start
   ```

   The agent will listen on `http://127.0.0.1:9100`.

## API

### POST /print

Send receipt data as JSON. The agent converts it to ESC/POS commands and prints.

**Request body** — structured receipt object (generated by `inv-receipt-generator.js`):

```json

  "companyName": "Tech Cross",
  "companyLogo": "logo.png",
  "companyAddress": "Unit 4, Navan Shopping Centre, Navan, Co. Meath, Ireland",
  "companyPhone": "046 905 9854",
  "receiptNumber": "20260428155415",
  "date": "28/04/2026 15:54:15",
  "items": [
    {
      "name": "iPhone 15 128GB",
      "quantity": 1,
      "unitPrice": 899.00,
      "discountedPrice": 899.00,
      "subtotal": 899.00,
      "isSecondHand": true,
      "serialNumber": "353456789012345"
    }
  ],
  "discountInfo": {
    "itemDiscounts": [],
    "orderDiscount": null
  },
  "totalAmount": 899.00,
  "paymentMethod": "card",
  "cashReceived": null,
  "changeGiven": null,
```

**Response:**


### GET /status

Check printer connectivity and view recent print logs.

**Response:**

```json
{
  "connected": true,
  "connectionType": "network",
  "printerConfig": { "type": "network", "host": "192.168.1.100", "port": 9100 },
  "usbFallback": { "vendorId": "0x0416", "productId": "0x5011" },
  "error": null,
  "recentLogs": []
}
```

## Connection Priority

1. **Network** — connects to the printer via TCP/IP (recommended for reliability)
2. **USB** — falls back to direct USB connection if network is unavailable

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `ECONNREFUSED` on network | Check printer IP and port in `config.json`, ensure printer is on the same network |
| USB device not found | Install libusb / Zadig driver, check vendor/product IDs with `lsusb` |
| Logo not printing | Verify `logoPath` points to a valid PNG file |
| Port already in use | Change `port` in `config.json` or stop the conflicting process |

## Architecture

```
Browser (POS page)
    │
    │  POST /print { receipt JSON }
    ▼
Print Agent (localhost:9100)
    │
    │  ESC/POS commands
    ▼
80mm Thermal Printer
```

The POS page in the browser sends receipt data to the Print Agent. If the agent is unreachable (not running or printer disconnected), the POS page falls back to `window.print()` for manual browser printing.
